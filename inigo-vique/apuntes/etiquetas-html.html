<body>


	<header>
		Contenido dentro del encabezado.  El <header> siempre tiene que estar en el <body>
		<h1>Es recomendable que todas las páginas necesitan un h1 o un title.</h1>
		También se puede meter, dentro del header la etiqueta <nav>, para colocar menú.
			* el <nav> no tiene porque ir dentro del <header>
			* Dentro de <nav> hay que meterle un <ul> e <li>
	</header>
	
	<main>
		La etiqueta <main>:  	Sólo puede haber 1.
								El contenido del documento que es susceptible de cambiar
								entre página y página del site.
								Selector de significado semántico.

		<article> y <section>:	Opiniones disputadas

		<section>:				Un main puede tener múltiples secciones.
								* una noticia
								* un formulario en la misma página

								TODAS LAS SECTION TIENEN QUE TENER SU PROPIO HEADER
								* Pueden tener su propio header y footer
								

		<article>:				Etiqueta parecida a section, con pequeñas diferencias.
								Por poner un ejemplo:
									Un grupo de noticias -> <section>
									Cada una de las noticias -> <article>

								Y cada <article> puede tener sus propias <section>

	</main>

	<footer>
		Es la parte del pie.  Tiene valor semántico.
			* a la gente le encanta los footers enormes
	</footer>

		Es preferible usar las estructuras semánticas antes que llenarlo todo de divs.

		No puede haber <h2> sin <h1>.  Tienen que seguir su orden.
