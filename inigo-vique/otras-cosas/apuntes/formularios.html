Formularios

Página 106 del libro.

Un formulario tiene que tener la siguiente estructura

			
<form action="#">
    <p>
        Usuario
        <input type="text" name="usuario" id="usuario">
    </p>
    <p>
        Contraseña
        <input type="password" name="password" id="password">
    </p>
    <p>
        <input type="submit" value="NombreBoton">
    </p>
</form>

Para estructurarlo hay que seguro de esta manera

    
<form action="#" method="post">
	<p>
	    <label for="usuario">Usuario</label>Usuario
	    <input type="text" name="usuario" id="usuario">
	</p>
	<p>
	    <label for="password">Contraseña</label>
	    <input type="password" name="password" id="password">
	</p>
	<p>
	    <input type="submit" value="NombreBoton">
	</p>
</form>

El <label for=""> es importante. Relaciona el nombre que estamos usando con el bloque de input en el que vamos a escribirlo, mediante la etiqueta "id" de éste.
Añadir el label, no cambia el estilo, a menos que toquemos el CSS.
Al clickar en un elemento con <label> nos lleva al input que lleva su id.

Si enviamos un formulario sin method POST, los datos que hemos introducido quedarán en la vista de navegador.

<form action="#" method="post">
Al enviar el formulario de esta manera, envía los datos pero no se muestran en la ruta.
<form action="#" method="get">
Al enviar datos de esta manera se muestran en la ruta. Es como dejarlo vacío.

Controles para usar en un formulario

Si no tiene ningún tipo es de tipo "text". Si tiene un tipo no conocido por el navegador, también será de tipo "text".

<text>
Bloque de texto
<password>
Contraseñas
<submit>
Es un botón. "value" es el texto que va a tener el botón en su interior. Envía el formulario. Si queremos cambiar de página una vez enviada es el servidor quien se encarga de ello.
<select>

Crea un desplegable en el que introducir opciones mediante <option>
option lleva un value, pero no es obligatorio. Si no se especifica valor envía el contenido en texto del option. Es preferible añadirlo aún así.
Añadir "selected" hace que la opción que lo lleva sea la que se muestre primero.

Ejemplo

                    	
<select name="sexo" id="sexo">
    <option value="hombre">Hombre</option>
    <option value="mujer">Mujer</option>
    <option selected value="otro">Otro</option>
</select>

Los select pueden tener la opción "multiple". Lo convierte en un cuadro de vista y permite elegir más de una opción pulsando la tecla Control.
La opción "size='2'" hace que en los select solo se muestren unas opciones al principio.
Dentro de los select también se pueden añadir optgroups, para agrupar el contenido en bloques.

<input type="checkbox">
Cuadro de selección con tick
<input type="radio">

Es un sistema un poco distinto a los otros. Se escriben una serie de radiobuttons con el mismo name pero distintos values. De esta manera, envía el dato de tipo "name" con uno de los valores del value.
Para marcar uno por defecto utilizamos el atributo checked.

Ejemplo

                    	
<label for="tipopremium">Premium</label>
<input type="radio" id="tipopremium" name="tiposuscriptor" value="premium">
<label for="tipogratuito">Gratuíto</label>
<input type="radio" id="tipogratuito" name="tiposuscriptor" value="gratuito" checked>
<input type="reset">
Es un botón para limpiar los campos. La funcionalidad existe pero no es habitual su uso.
<input type="button">
Es un botón, pero sin ningún tipo de utilidad en principio, porque el de envío es el "submit".
<input type="file">

Selecciona un archivo para enviarlo. Al seleccionarlo no lo envía, simplemente lo relaciona con el navegador. Los ficheros hay que enviarlos por tipo POST.
También para enviarlo hay que añadirle un enctype al tipo de formulario, si no lo enviaría como texto, y no iría bien.
El enctype tiene que ser de tipo "multipart/form-data". 

                		
<form action="#" method="post" enctype="multipart/form-data"></form>
<textarea name="" id="" cols="30" rows="10">
Es un área de texto más grande de lo habitual. En algunos navegadores permite ampliarla sin problemas. Es la única etiqueta que tiene apertura y cierre.
Normalmente suele poner un tipo de letra monospace por defecto. Podemos cambiar el tipo de letra mediante CSS.
Permite usar placeholder para las pistas.
<input type="hidden" name="ocultos" value="x">
El dato si que se envía pero el usuario no tiene manera de modificarlo.
Son datos que se introducen automáticamente o se pueden modificar mediante Javascript.
<fieldset></fieldset>
Encuadra los datos en un bloque. Sólo visual. Dentro se le añade la etiqueta <legend></legend> Para ponerle nombre a ese grupo de datos.

Existen nuvos input type's en HTML5.

week, url, time, month, email, ...

Es mejor usarlos. Los navegadores que no tienen capacidad para mostrarlo les muestra un cuadro de texto y lo pueden añadir sin problema. Así es más semántico.
Es bueno para los navegadores móviles, porque crean teclados especiales para cada tipo de input (si el navegador es bueno).

Otras cosas

El regex se le añade en la opción "pattern". Le dirá al navegador de que tamaño y que contenido a de tener el texto que introducimos.
"required". Obliga a que el dato haya que introducirlo.
"placeholder". Es un texto que aparece dentro de los input text. Si está vacío aparece un poco más gris. Al introducir cualquier texto desaparece pero si está vacío vuelve a aparecer.
minlength. Obliga a que el campo se llene con al menos X caracteres. Es una manera más sencilla de introducir un REGEX, aunque no funciona en EDGE.
